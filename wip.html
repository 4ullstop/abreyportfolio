Scene (Working Title)

  • Platform: PC
  • Software: GCC, Raylib, Visual Studio Code
  • Creation Length: 2 months
  • Link to GitHub Source Code
  • Regular updates on LinkedIn

Scene (Working Title) is a first person puzzler in which your main interaction consists of solving puzzles of buttons which consist of various rules. The other main interaction the player will have is the ability to play between two different windows at once. Through the utilization of multiple executables and shared memory, the player will have to navigate between two different worlds at the same time with challenges spread across both windows.

Responsibilities

  • Develop C code using GCC, Visual Studio Code and Raylib Graphics Library
  • Develop a personalized camera system
  • Develop collision system and collision response code
  • Create a workflow for loading and unloading models from render
  • Develop an interaction system using a custom collision based raycast
  • Develop a system to utilize two windows and shared memory
  • Design and develop button selection puzzle mechanic

Shared Memory

Given the unusual design of this game, I had to devise some interesting techniques to achieve the multi-window gameplay. In order to have two windows communicate with each other, each process needed to open up a portion of memory to share between each other. Then through this memory, location of the puzzle's cursor, the state of the other game, and other factors that determine gameplay.

Collision Detection

This project has started with the creation of a custom collision detection and response algorithm which I spent a couple of days studying and translating from the article Improved Collision detection and Response by Kasper Fauerby.

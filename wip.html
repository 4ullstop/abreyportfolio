Collision Detection

This project has started with the creation of a custom collision detection and response algorithm which I spent a couple of days studying and translating from the article Improved Collision detection and Response by Kasper Fauerby.

The collision detection has a couple of steps:

  1. First we construct a triangle from the given vertices of the mesh
  2. Then we find the time at which the sphere could intersect with the plane
  3. We then check if the collision occurs inside of the triangle or whether it's on the edge or vertice
  4. If it ends up colliding with the edge or vertice we do a sweep across them

Doing these calculations tells us whether or not we will intersect and at what point and distance

Next we run our collision response algorithm if the detection algorithm discovers a potential hit.

  1. We start by moving our player very close to the intersection point
  2. We then calculate what is known as a sliding plane (a very long plane that is perpendicular to the triangle the player hit)
  3. We then adjust and project the player's velocity according to this sliding plane

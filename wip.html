Memory Pools

Using an approach known as memory pools, I allocate a large chunk of memory up front, and then use the space to push any struct I need. The memory is then released at the end of the program as opposed to releasing as according to the lifetime of a specific object. This in turn makes memory management simple to use.

I took things a step further when implementing my 'redo' button using a fixed size pool. Whenever the player presses a movement button, their location is recorded in a linked list which is entirely allocated at the start of the program. When adding to the list, we check another list of 'freed' locations which if there are any we grab from and use to 'allocate' a new node. When the player presses the 'redo' button and the state is set to whichever state existed in the most recent node, we then 'free' the node by placing it in the list of 'freed' locations.

Programming Style

As observed from previous projects, keeping structure and style for code is incredibly important. With that being said, I find that the best way to program for my solo projects is to keep everything functional and programming with the purpose of compressing only when absolutely necessary. Through this, I've found a few key benefits:

First of which being, by not spending time trying to immediately comparmentalize things into functions, I can focus solely on writing code that works. This also makes my code incredibly flexible which allows me change things with ease when I iterate on a design idea.

As the most recent example, when developing two entities, the player and the ball, I decided I wanted to switch things up and make it so the player is the ball/has the same properties as the ball in terms of movement. In my structure, ball_entity was a child of entity with player being this type. There were a couple of functions which the ball used for its movement which I needed to swap to allow the player to have. This was a simple 10 minute fix with a couple of swaps of data types and suddenly I had the characteristics I was looking for.
